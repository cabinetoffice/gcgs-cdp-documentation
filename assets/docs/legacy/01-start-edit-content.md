# Start edit

**Start edit** → Edit content → Start edit navigation → Edit navigation → Preview content → Request review

This guide is for:
* Content editors - creating or updating site content
* Testers  - creating test content to verify the editing workflow

It shows how to
* Navigate to the project repository
* Open the repository Code view
* Select the correct base branch for your role


## Step 1 – Open this guide in a new tab

Keep this guide open in another browser tab so you can follow the instructions while working in GitHub.


## Step 2 - Switch to the `Code` tab

Select the Code tab at the top of the repository to view the repository files and branches.

   <img src="../images/01-select-code-tab.png" width="300">


## Step 3 – Select the correct base branch

> [!WARNING]
>
> Select the correct base branch for your role.
>
> | Role | Base branch |
> |-----|-------------|
> | Content editors | `main` |
> | Testers | `staging` |
>
> Testers **must always create branches from `staging`**.

Use the branch dropdown to select the base branch you need to work from.

<img src="../images/01a-select-base-branch.png" width="300">
